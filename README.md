# Лабораторна робота №2
## Дослідження роботи з компонентом Fragment

**Студент:** Попов Владислав ІП-35 **Варіант:** 14

### Мета роботи
Дослідити створення та взаємодію з компонентом Fragment у межах однієї Activity та набути практичних навичок використання фрагментів для побудови інтерфейсу користувача.

---

## Завдання
Розробити Android-застосунок, який містить інтерфейс, побудований з кількох Fragment.  
Перший фрагмент використовується для введення даних, другий — для відображення результату.  
Передача та збереження стану реалізовані за допомогою ViewModel.

---

## Опис застосунку
У застосунку реалізовано вибір автора книги та року видання.

### Перший фрагмент (`InputFragment`)
Містить:
- `Spinner` для вибору автора;
- `RadioGroup` для вибору року видання;
- кнопку **OK** для підтвердження введення.

### Другий фрагмент (`ResultFragment`)
Містить:
- `TextView` для відображення результату;
- кнопку **Cancel**, яка:
  - повертає користувача до першого фрагмента;
  - очищає введені дані.

---

## Структура застосунку
Застосунок складається з:
- `MainActivity` — контейнер для фрагментів;
- `InputFragment` — фрагмент введення даних;
- `ResultFragment` — фрагмент відображення результату;
- `SharedViewModel` — спільна ViewModel для передачі та збереження даних між фрагментами.

---

## Принцип роботи
1. Після запуску застосунку відкривається `InputFragment`.
2. Користувач обирає автора та рік видання.
3. Після натискання кнопки **OK** вибрані дані передаються через `SharedViewModel`.
4. Відкривається `ResultFragment`, у якому відображається результат.
5. Після натискання кнопки **Cancel** другий фрагмент закривається, а форма у першому фрагменті очищується.

## Висновок
У ході виконання лабораторної роботи було створено Android-застосунок з одним Activity та двома Fragment.  
Було досліджено життєвий цикл фрагментів, способи їх створення та взаємодії.  
Передача даних між фрагментами реалізована за допомогою `ViewModel`, що дозволяє коректно зберігати стан даних і організувати взаємодію між компонентами в межах однієї Activity.
